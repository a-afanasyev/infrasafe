# Безопасность API

## Аутентификация

API использует JWT (JSON Web Tokens) для аутентификации:

1. **Access Token**
   - Короткий срок жизни (15 минут)
   - Передается в заголовке: `Authorization: Bearer <token>`
   - Используется для всех защищенных запросов

2. **Refresh Token**
   - Длительный срок жизни (7 дней)
   - Используется только для обновления access token
   - Хранится в безопасном месте

## Авторизация

Система использует RBAC (Role Based Access Control):

1. **Роли**
   - admin - полный доступ
   - user - базовый доступ
   - viewer - только чтение

2. **Разрешения**
   - create - создание ресурсов
   - read - чтение ресурсов
   - update - обновление ресурсов
   - delete - удаление ресурсов

## Защита от атак

1. **Rate Limiting**
   - 1000 запросов в час для авторизованных пользователей
   - 100 запросов в час для неавторизованных
   - Отслеживание по IP и токену

2. **CORS**
   - Строгая политика источников
   - Whitelist разрешенных доменов
   - Проверка Origin заголовка

3. **Защита от инъекций**
   - Параметризованные запросы
   - Валидация входных данных
   - Экранирование специальных символов

4. **XSS Protection**
   - Content Security Policy
   - Sanitization входных данных
   - HttpOnly cookies

## Обработка чувствительных данных

1. **Шифрование**
   - TLS 1.3 для передачи
   - AES-256 для хранения
   - Bcrypt для паролей

2. **Логирование**
   - Без чувствительных данных
   - Ротация логов
   - Аудит действий

## Мониторинг безопасности

1. **Отслеживание**
   - Подозрительная активность
   - Попытки взлома
   - Аномальное поведение

2. **Оповещения**
   - Критические события
   - Превышение лимитов
   - Сбои безопасности

## Рекомендации для клиентов

1. **Хранение токенов**
   - Access token в памяти
   - Refresh token в secure storage
   - Очистка при выходе

2. **Обработка ошибок**
   - 401 - обновление токена
   - 403 - проверка прав
   - 429 - ожидание и повтор

3. **Безопасные запросы**
   - HTTPS обязателен
   - Валидация сертификатов
   - Проверка ответов
