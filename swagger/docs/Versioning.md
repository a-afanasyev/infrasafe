# Версионирование API

## Общие принципы

API использует семантическое версионирование (MAJOR.MINOR.PATCH):
- MAJOR - несовместимые изменения API
- MINOR - добавление новой функциональности с обратной совместимостью
- PATCH - исправления ошибок с обратной совместимостью

## Указание версии

Версия API указывается в:
1. URL пути: `/api/v1/resource`
2. Заголовке Accept: `application/vnd.infrasafe.v1+json`

## Поддерживаемые версии

- v1 - текущая стабильная версия
- v2 - в разработке (бета)

## Жизненный цикл версий

1. Стабильная версия поддерживается минимум 12 месяцев
2. Устаревшие версии помечаются заголовком Deprecation
3. О прекращении поддержки сообщается за 6 месяцев

## Обратная совместимость

Изменения, сохраняющие обратную совместимость:
- Добавление новых эндпоинтов
- Добавление необязательных параметров
- Добавление полей в ответы
- Добавление новых значений в enum

Изменения, нарушающие обратную совместимость:
- Удаление или переименование эндпоинтов
- Изменение типов данных
- Добавление обязательных параметров
- Удаление полей из ответов

## Миграция между версиями

При выпуске новой MAJOR версии:
1. Предоставляется руководство по миграции
2. Обе версии работают параллельно минимум 6 месяцев
3. Старая версия помечается как устаревшая
4. Клиенты получают уведомления о необходимости обновления

## Тестирование версий

Каждая версия API имеет:
1. Отдельный набор тестов
2. Документацию изменений
3. Примеры использования
4. Инструменты миграции
