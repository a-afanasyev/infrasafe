## 🛡️ Патентная защита

### Что это значит:

Apache 2.0 включает **явную патентную лицензию**:

1. **Вы получаете права** на все патенты, необходимые для использования InfraSafe
2. **Патентное возмездие** - если кто-то подает патентный иск, они теряют права на использование
3. **Защита пользователей** - конечные пользователи защищены от патентных претензий

### Пример:
```
Компания А использует InfraSafe → Получает патентную лицензию
Компания А подает патентный иск → Теряет права на InfraSafe
```

## 🇺🇿 Специфика для Узбекистана

### Преимущества Apache 2.0 для госсектора:

1. **Прозрачность**
   - Открытый код для аудита безопасности
   - Проверка на backdoors и уязвимости
   - Соответствие требованиям информационной безопасности

2. **Независимость**
   - Нет привязки к иностранным вендорам
   - Развертывание на местных серверах
   - Полный контроль над данными

3. **Экономия**
   - Нет лицензионных платежей
   - Оплата только за услуги
   - Масштабирование без дополнительных затрат

4. **Юридическая чистота**
   - Международно признанная лицензия
   - Четкие права и обязанности
   - Подходит для госконтрактов

## 📞 Часто задаваемые вопросы

### Q: Можем ли мы продавать InfraSafe?
**A:** Да, вы можете продавать услуги на базе InfraSafe (внедрение, поддержка, кастомизация, хостинг).

### Q: Нужно ли платить лицензионные отчисления?
**A:** Нет, Apache 2.0 не требует royalty или лицензионных платежей.

### Q: Можем ли мы закрыть исходный код наших модификаций?
**A:** Да, Apache 2.0 позволяет создавать закрытые производные работы.

### Q: Что если мы найдем баг и исправим его?
**A:** Вы можете использовать исправление приватно. Если захотите поделиться - будем рады pull request!

### Q: Подходит ли для государственных тендеров?
**A:** Да, Apache 2.0 - это международно признанная open source лицензия, подходящая для госконтрактов.

### Q: Можем ли мы использовать название "InfraSafe"?
**A:** Для внутреннего использования - да. Для коммерческого ребрендинга - лучше использовать свое название.

### Q: Что делать при получении патентного иска?
**A:** Apache 2.0 включает Patent Retaliation Clause - истец теряет права на использование ПО.
