# Промпт для агента: Полный аудит проекта Infrasafe

Скопируй этот промпт в Claude Code (или другого агента с доступом к файловой системе проекта).

---

## ПРОМПТ (начало)

Ты — технический аудитор. Твоя задача — провести полный анализ проекта и сохранить результат в один файл `PROJECT_CONTEXT.md`, который будет использоваться как источник истины для другого AI-ассистента.

Работай методично. Не пропускай шаги. Если что-то неясно — фиксируй как вопрос, а не додумывай.

---

### ШАГ 1. Разведка структуры

```
find . -maxdepth 4 -type f | grep -v node_modules | grep -v .git | grep -v __pycache__ | grep -v .pyc | head -500
```

Также выполни:
```
find . -name "docker-compose*" -o -name "Dockerfile*" -o -name "Makefile" -o -name ".env*" -o -name "*.toml" -o -name "*.cfg" -o -name "requirements*.txt" -o -name "package.json" -o -name "pyproject.toml" | grep -v node_modules
```

Зафиксируй полное дерево каталогов (до 4 уровней).

---

### ШАГ 2. Стек и зависимости

Для каждого найденного файла зависимостей (`requirements.txt`, `package.json`, `pyproject.toml`, `go.mod`, `Cargo.toml` и т.д.):
- Прочитай содержимое
- Зафиксируй все зависимости с версиями
- Определи язык/фреймворк каждого сервиса

Для Docker:
- Прочитай все `Dockerfile` и `docker-compose*.yml`
- Зафиксируй: какие сервисы, какие образы, какие порты, какие volumes, какие сети
- Найди переменные окружения (`.env`, `.env.example`, `env` в compose)

---

### ШАГ 3. Архитектура и связи между компонентами

Определи и опиши:
1. **Список сервисов/модулей** — что делает каждый
2. **Точки входа** — main-файлы, entrypoint'ы
3. **API-эндпоинты** — найди роутеры, контроллеры, обработчики (grep по `@app.route`, `router`, `@api`, `endpoint`, `handler` и т.д.)
4. **Взаимодействие между сервисами** — кто куда обращается (HTTP, MQTT, gRPC, очереди)
5. **Базы данных** — какие используются, где конфигурация, есть ли миграции
6. **Брокеры сообщений** — MQTT-топики, подписки, паблишеры (grep по `publish`, `subscribe`, `topic`)
7. **Внешние интеграции** — API третьих сторон, вебхуки

Выполни:
```
grep -rn "mqtt" --include="*.py" --include="*.js" --include="*.ts" --include="*.go" --include="*.yaml" --include="*.yml" . | grep -v node_modules | head -50
grep -rn "influx\|influxdb" --include="*.py" --include="*.js" --include="*.ts" --include="*.go" --include="*.yaml" --include="*.yml" . | grep -v node_modules | head -50
grep -rn "postgres\|postgresql\|psycopg\|sqlalchemy" --include="*.py" --include="*.js" --include="*.ts" --include="*.go" --include="*.yaml" --include="*.yml" . | grep -v node_modules | head -50
```

---

### ШАГ 4. Модели данных

- Найди ORM-модели, схемы БД, миграции
- Прочитай SQL-миграции если есть
- Зафиксируй структуру таблиц/коллекций
- Для InfluxDB — какие measurements, какие теги, какие поля

```
find . -path "*/models*" -o -path "*/schema*" -o -path "*/migration*" -o -path "*/alembic*" | grep -v node_modules | grep -v .git
```

---

### ШАГ 5. Конфигурация и деплой

- Как запускается проект (инструкции, скрипты, Makefile)
- CI/CD — есть ли `.github/workflows`, `gitlab-ci.yml`, `Jenkinsfile`
- Конфигурация окружений (dev/staging/prod)
- Секреты — какие нужны (НЕ их значения, только названия)
- Где и как деплоится (VPS, облако, k8s)

---

### ШАГ 6. Существующая документация

```
find . -name "*.md" -o -name "*.rst" -o -name "*.txt" -name "README*" -o -name "CHANGELOG*" -o -name "TODO*" -o -name "CONTRIBUTING*" | grep -v node_modules | grep -v .git
```

Для каждого найденного документа:
- Прочитай содержимое
- Оцени актуальность (совпадает ли с реальным кодом)
- Отметь что устарело или отсутствует

---

### ШАГ 7. Проблемы и технический долг

- Есть ли TODO/FIXME/HACK в коде:
```
grep -rn "TODO\|FIXME\|HACK\|XXX\|WORKAROUND" --include="*.py" --include="*.js" --include="*.ts" --include="*.go" . | grep -v node_modules | head -50
```
- Есть ли неиспользуемые файлы, мёртвый код
- Есть ли тесты? Какое покрытие?
```
find . -name "test_*" -o -name "*_test.*" -o -name "*.test.*" -o -name "*.spec.*" | grep -v node_modules | head -30
```
