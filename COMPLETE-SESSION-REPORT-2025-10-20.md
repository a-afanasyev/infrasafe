## 2. ИСПРАВЛЕННЫЕ ПРОБЛЕМЫ

### Проблема #1: Генератор не подключался к API ❌ → ✅

**Симптомы**: `Error: connect ECONNREFUSED 127.0.0.1:3000`

**Причина**: Контейнер использовал `localhost:3000`

**Решение**: 
- `API_BASE_URL=http://host.docker.internal:3000/api`
- `--add-host=host.docker.internal:host-gateway`

---

### Проблема #2: Неверный пароль для API ❌ → ✅

**Симптомы**: `401 Unauthorized`

**Причина**: Пароль `admin123` вместо `Admin123`

**Решение**: Обновлён `.env`: `API_PASSWORD=Admin123`

---

### Проблема #3: Парсинг ответа API ❌ → ✅

**Симптомы**: Генератор не находил здания с контроллерами

**Причина**: API возвращает `{data: [...]}`, код ожидал массив

**Решение**: `const buildings = data?.data || data;`

---

### Проблема #4: Удаление зданий с контроллерами ❌ → ✅

**Симптомы**: Операция молчаливо не выполнялась

**Причина**: Frontend не обрабатывал ошибку `BUILDING_HAS_CONTROLLERS`

**Решение**: Реализованы информативные диалоги + каскадное удаление

---

### Проблема #5: Методы слоёв не реализованы ❌ → ✅

**Симптомы**: Ошибка в консоли при включении контроллеров/алертов

**Причина**: Методы `loadControllers()` и `loadAlerts()` отсутствовали

**Решение**: Реализованы все 6 недостающих методов

---

### Проблема #6: Координаты трансформаторов отсутствовали ❌ → ✅

**Симптомы**: Трансформаторы не отображались на карте

**Причина**: 
- Таблица `transformers` не имела полей `latitude`/`longitude`
- Model не возвращал координаты

**Решение**:
```sql
ALTER TABLE transformers 
  ADD COLUMN latitude NUMERIC(9,6),
  ADD COLUMN longitude NUMERIC(9,6),
  ADD COLUMN geom geometry(POINT, 4326);

UPDATE transformers SET ... -- 16 записей обновлено
```
```javascript
// src/models/Transformer.js
this.latitude = data.latitude;
this.longitude = data.longitude;
```

---

### Проблема #7: Координаты возвращались как строки ❌ → ✅

**Симптомы**: API возвращает `"41.311081"` вместо `41.311081`

**Причина**: PostgreSQL NUMERIC конвертируется в string в JSON

**Решение**: Добавлен `parseFloat()` во всех методах создания маркеров

## 9. ИЗВЕСТНЫЕ ОГРАНИЧЕНИЯ

### Генератор
- Нет UI для просмотра истории генерации
- Нет возможности изменить cron через UI
- Данные сбрасываются при перезапуске контейнера (volume монтируется к файлу)

### Админка
- Каскадное удаление выполняется последовательно (может быть медленным)
- Нет предпросмотра полного дерева зависимостей
- Нет восстановления удалённых данных

### Карта
- Линии ЛЭП: счётчик работает, визуализация требует координат трансформаторов
- Источники воды/тепла: таблицы БД пусты
- Popup требуют клика или двойного клика (не hover)

## 10. РЕКОМЕНДАЦИИ

### Краткосрочные (1-2 дня)

1. **Исправить volume генератора**
   ```yaml
   # Вместо:
   - generator_data:/app/generator-data.json
   # Использовать:
   - generator_data:/app/data
   ```

2. **Заполнить таблицы источников**
   ```sql
   INSERT INTO cold_water_sources (id, name, address, latitude, longitude, ...)
   VALUES (...);
   
   INSERT INTO heat_sources (id, name, address, latitude, longitude, ...)
   VALUES (...);
   ```

3. **Визуализация линий ЛЭП**
   - Получить координаты трансформаторов
   - Использовать `L.polyline([coord1, coord2], {...})`

### Среднесрочные (1-2 недели)

1. **Backend endpoint для каскадного удаления**
   ```javascript
   DELETE /api/buildings/:id?cascade=true
   ```

2. **Soft delete для возможности восстановления**
   ```sql
   ALTER TABLE buildings ADD COLUMN deleted_at TIMESTAMP;
   ALTER TABLE controllers ADD COLUMN deleted_at TIMESTAMP;
   ```

3. **UI улучшения генератора**
   - История генерации
   - Настройка cron через UI
   - Экспорт/импорт конфигурации

### Долгосрочные (1-2 месяца)

1. **Clustering маркеров**
   - Использовать leaflet-markercluster
   - При zoom out группировать объекты

2. **Фильтры на карте**
   - Реализовать фильтрацию по статусу
   - Фильтр по загрузке трансформаторов
   - Фильтр по типу водоснабжения

3. **Система аудита удалений**
   - Лог всех операций удаления
   - Возможность восстановления
   - Отчёты по изменениям
