# Интеграция с API

## Базовый URL

API клиент настроен на использование базового URL:
- По умолчанию: `/api`
- Можно переопределить через `window.BACKEND_URL` в HTML

## Обработка ошибок

### Улучшенная обработка ошибок включает:

1. **HTTP ошибки (401, 403, 404, 500 и т.д.)**
   - Автоматическое извлечение сообщения об ошибке из JSON ответа
   - Отображение понятных сообщений пользователю

2. **Ошибки парсинга JSON**
   - Проверка на пустой ответ
   - Детальные сообщения об ошибках парсинга

3. **Ошибки сети**
   - Определение ошибок подключения к серверу
   - Понятные сообщения для пользователя

4. **Валидация данных**
   - Проверка формата ответа (массив или объект с data)
   - Проверка наличия обязательных полей

## Toast уведомления

Все ошибки отображаются через систему Toast уведомлений:
- Красный цвет для ошибок
- Желтый для предупреждений
- Зеленый для успешных операций
- Синий для информационных сообщений

## Автоматическое обновление

Реализована система автоматического обновления данных:
- Настраиваемый интервал обновления (30 сек, 1 мин, 5 мин, 10 мин)
- Кнопка ручного обновления
- Отображение времени последнего обновления

## Статусы обработки

Функция `loadData()` возвращает:
- `true` - при успешной загрузке данных
- `false` - при ошибке загрузки

Это позволяет обрабатывать результат загрузки и принимать соответствующие действия.
