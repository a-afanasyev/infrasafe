## ⚠️ ВЫЯВЛЕННОЕ ДУБЛИРОВАНИЕ

### Проблема #1: Дублирование Типов Линий

```
ЭЛЕКТРИЧЕСТВО:
├─ Вкладка "Линии"               → 10 линий из lines
└─ Вкладка "Линии на карте"      → 1 линия из infrastructure_lines
   └─ Кнопка "Создать линию электропередач" ← ДУБЛИРУЕТ!

ХВС/ГВС (Водоснабжение):
├─ Вкладка "Линии ХВС/ГВС"       → 0 линий из water_lines
└─ Вкладка "Линии на карте"      → 2 линии из infrastructure_lines
   └─ Кнопки "Создать ХВС/ГВС"   ← ДУБЛИРУЕТ!
```

**Результат:** Админ может создать линию электричества в **ДВУХ разных местах**, что приведёт к:
- Путанице: где какие линии?
- Дублированию данных в разных таблицах
- Несогласованности

---

| Таблица | Формат | Поддержка изломов |
|---------|--------|-------------------|

| `water_lines` | ❌ start/end координаты | ❌ Нет |
| `infrastructure_lines` | ✅ main_path | ✅ Да |

**Результат:** 
- `water_lines` отстаёт по функционалу
- Разный UX для разных типов линий
- Непоследовательность

---

### Проблема #3: Разные Редакторы

| Вкладка | Редактор | Возможности |

| Линии | InfrastructureLineEditor | Карта, изломы, ответвления ✅ |

**Результат:**

### Этап 1: Миграция данных из infrastructure_lines

```sql
-- 1. Мигрировать линию электричества в lines
INSERT INTO lines (name, voltage_kv, main_path, branches, cable_type, commissioning_year)
SELECT name, voltage_kv, main_path, branches, cable_type, commissioning_year
FROM infrastructure_lines
WHERE line_type = 'electricity';

-- 2. Обновить water_lines (добавить поля)
ALTER TABLE water_lines 
ADD COLUMN main_path JSONB,
ADD COLUMN branches JSONB DEFAULT '[]'::jsonb;

-- 3. Мигрировать линии воды в water_lines
INSERT INTO water_lines (name, description, line_type, main_path, branches, diameter_mm, material)
SELECT name, description, 
       CASE 
         WHEN line_type = 'cold_water' THEN 'ХВС'
         WHEN line_type = 'hot_water' THEN 'ГВС'
       END,
       main_path, branches, diameter_mm, material
FROM infrastructure_lines
WHERE line_type IN ('cold_water', 'hot_water');

-- 4. Удалить infrastructure_lines
DROP TABLE infrastructure_lines CASCADE;
```
