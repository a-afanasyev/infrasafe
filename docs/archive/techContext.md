# InfraSafe - Технический контекст

## Архитектура приложения

### Бэкенд структура
```
src/
├── config/                   # Конфигурация приложения
│   ├── app.js               # Основные настройки
│   └── database.js          # Настройки подключения к БД
├── controllers/             # Контроллеры API
├── middleware/              # Промежуточное ПО
├── models/                 # Модели данных
├── routes/                 # Маршруты API
├── services/               # Бизнес-логика (сервисный слой)
├── utils/                  # Утилиты и помощники
└── server.js               # Точка входа приложения
```

### Ключевые компоненты

#### Сервисный слой (95% завершён)
- **buildingService.js** - CRUD операции зданий с кэшированием
- **controllerService.js** - Управление контроллерами IoT
- **metricService.js** - Обработка метрик и телеметрии
- **authService.js** - Авторизация и безопасность
- **alertService.js** - Система уведомлений

#### База данных
- **PostgreSQL 15+** с полной нормализацией
- Миграции в `database/init/`
- Модели: Building, Controller, Metric, Alert, etc.

#### API уровень
- RESTful API с Swagger документацией
- JWT авторизация
- Валидация данных
- Обработка ошибок
- Rate limiting

### Производительность и надёжность
- Кэширование с TTL
- Circuit breaker паттерн
- Логирование операций
- Мониторинг метрик
- Автоматическое определение статусов

### Контейнеризация
- **Docker Compose** для оркестрации
- Отдельные контейнеры: фронтенд, бэкенд, база данных
- Nginx как reverse proxy
- Настройки для dev/prod окружений
