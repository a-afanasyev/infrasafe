# T003: План автоматического определения порта

## 📋 Обзор

**Цель:** Создать универсальную систему тестирования, которая автоматически определяет рабочий порт (3000 или 8080) и использует его для всех тестов.

**Проблема:** Различные тесты используют разные порты, что приводит к конфликтам и ошибкам.

**Решение:** Универсальная утилита для автоматического определения порта с fallback механизмами.

## 🎯 Требования

### Функциональные требования
1. **Автоматическое определение порта:**
   - Проверка доступности порта 3000
   - Проверка доступности порта 8080
   - Выбор первого доступного порта
   - Fallback на порт 3000 по умолчанию

2. **Универсальность:**
   - Работа с bash скриптами
   - Работа с Jest тестами
   - Работа с нагрузочными тестами
   - Работа с smoke тестами

3. **Надежность:**
   - Timeout для проверки портов
   - Логирование процесса определения
   - Обработка ошибок
   - Fallback механизмы

### Нефункциональные требования
- **Время отклика:** < 5 секунд на определение порта
- **Надежность:** 99% успешного определения
- **Совместимость:** Работает на macOS, Linux, Windows
- **Простота:** Минимальные изменения в существующих тестах

## 🏗️ Архитектура решения

### 1. Утилита определения порта

#### Bash утилита (`tests/utils/port-detector.sh`)
```bash
#!/bin/bash
# Автоматическое определение рабочего порта

detect_api_port() {
    local port_3000_available=false
    local port_8080_available=false
    
    # Проверка порта 3000
    if curl -s --connect-timeout 3 http://localhost:3000/api/health > /dev/null 2>&1; then
        port_3000_available=true
        echo "3000"
        return 0
    fi
    
    # Проверка порта 8080
    if curl -s --connect-timeout 3 http://localhost:8080/api/health > /dev/null 2>&1; then
        port_8080_available=true
        echo "8080"
        return 0
    fi
    
    # Fallback на порт 3000
    echo "3000"
    return 1
}

export API_PORT=$(detect_api_port)
export API_URL="http://localhost:$API_PORT"
```

#### JavaScript утилита (`tests/utils/port-detector.js`)
```javascript
// Автоматическое определение порта для Jest тестов
const axios = require('axios');

async function detectApiPort() {
    const ports = [3000, 8080];
    const timeout = 3000;
    
    for (const port of ports) {
        try {
            const response = await axios.get(`http://localhost:${port}/api/health`, {
                timeout: timeout
            });
            if (response.status === 200) {
                return port;
            }
        } catch (error) {
            continue;
        }
    }
    
    // Fallback на порт 3000
    return 3000;
}

module.exports = { detectApiPort };
```

#### Bash тесты
```bash

```

#### Jest тесты
```javascript
// В setup.js
const { detectApiPort } = require('./utils/port-detector');

beforeAll(async () => {
    const port = await detectApiPort();
    process.env.API_PORT = port;
    process.env.API_URL = `http://localhost:${port}`;
});
```
