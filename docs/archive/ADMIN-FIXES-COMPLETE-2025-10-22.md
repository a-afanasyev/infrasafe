### 1. Исправление редактора координат (`admin-coordinate-editor.js`)
**Проблема**: Некорректная обработка параметров при вызове `openCoordinateEditor` из inline-обработчиков в `admin.html`.

**Решение**:
- Модифицирована функция `openCoordinateEditor` для поддержки двух вариантов вызова:
  - `openCoordinateEditor(objectType, objectId, callback)` - для inline-обработчиков
  - `openCoordinateEditor(objectType, objectId, latitude, longitude, objectName, onSave)` - полный вызов

**Код исправления** (строки 367-397 в `admin-coordinate-editor.js`):
```javascript
function openCoordinateEditor(objectType, objectId, latitudeOrCallback, longitude, objectName, onSave) {
    // Определяем, какой вариант вызова используется
    let actualLatitude, actualLongitude, actualObjectName, actualOnSave;
    
    if (typeof latitudeOrCallback === 'function') {
        // Вариант 1: inline callback
        actualLatitude = null;
        actualLongitude = null;
        actualObjectName = null;
        actualOnSave = latitudeOrCallback;
    } else {
        // Вариант 2: полный вызов
        actualLatitude = latitudeOrCallback;
        actualLongitude = longitude;
        actualObjectName = objectName;
        actualOnSave = onSave;
    }
    
    const editor = new CoordinateEditor({
        objectType,
        objectId,
        latitude: actualLatitude,
        longitude: actualLongitude,
        objectName: actualObjectName,
        onSave: actualOnSave
    });
    
    editor.show();
}
```

### 2. Исправление редактора линий (`infrastructure-line-editor.js`)

#### 2.1. Устранение конфликта ID элементов
**Проблема**: Методы `validateLine()` и `saveLine()` использовали `document.getElementById('line-name')`, который мог захватывать элементы из других (скрытых) частей DOM.

**Решение**:
- Все запросы элементов теперь выполняются внутри контекста модального окна
- Используется `modal.querySelector()` вместо `document.getElementById()`

**Код исправления** (строки 638-682 в `infrastructure-line-editor.js`):
```javascript
validateLine() {
    const errors = [];

    // Ищем элементы только внутри текущего модального окна
    const modal = document.getElementById('infrastructure-line-editor-modal');
    if (!modal) {
        console.error('❌ Модальное окно редактора линий не найдено в DOM');
        errors.push('Ошибка: модальное окно не найдено');
        return errors;
    }

    // Проверка названия - ищем внутри модального окна
    const nameElement = modal.querySelector('#line-name');
    if (!nameElement) {
        console.error('❌ Элемент line-name не найден внутри модального окна');
        errors.push('Ошибка: элемент названия линии не найден');
        return errors;
    }
    
    const name = nameElement.value.trim();
    console.log('🔍 Валидация названия линии:', { 
        value: nameElement.value, 
        trimmed: name, 
        isEmpty: !name,
        modalId: modal.id
    });
    
    if (!name) {
        errors.push('Название линии обязательно');
    }
    // ... остальная валидация
}
```

#### 2.2. Предотвращение дублирования модальных окон
**Проблема**: При многократном открытии редактора могли создаваться дубликаты модальных окон с одинаковыми ID.

**Решение**:
- Метод `show()` теперь удаляет все существующие модальные окна перед созданием нового
- Метод `close()` удаляет все экземпляры модального окна

**Код исправления** (строки 79-101 в `infrastructure-line-editor.js`):
```javascript
show() {
    // Удаляем все существующие модальные окна перед созданием нового
    const existingModals = document.querySelectorAll('#infrastructure-line-editor-modal');
    existingModals.forEach(modal => {
        console.log('⚠️ Удаляем существующее модальное окно редактора линий');
        modal.remove();
    });
    
    const modalHTML = this.createModalHTML();
    const modalContainer = document.createElement('div');
    modalContainer.id = 'infrastructure-line-editor-modal';
    modalContainer.innerHTML = modalHTML;
    document.body.appendChild(modalContainer);

    // Инициализируем обработчики после добавления в DOM
    setTimeout(() => {
        this.attachEventHandlers();
        this.initializeMap();
        this.renderExistingLine();
    }, 100);
}
```

#### 2.3. Исправление отрисовки существующих линий
**Проблема**: При редактировании существующей линии точки добавлялись дважды - при инициализации и при отрисовке.

**Решение**:
- Метод `renderExistingLine()` больше не добавляет точки в `this.mainPath`
- Точки загружаются только при инициализации класса
- При отрисовке создаются только маркеры и линии на основе существующих данных
