## Возможные причины

### 1. Защита от случайного удаления
Система может иметь защиту от удаления зданий с активными контроллерами:
- Frontend показывает диалог, но backend отклоняет запрос
- Нет видимого уведомления об ошибке для пользователя

### 2. Проблема с каскадным удалением
Возможно backend пытается удалить связанные данные, но:
- FK constraint блокирует удаление из-за связанных метрик
- Транзакция откатывается без уведомления пользователя

### 3. Проблема с правами доступа
- Пользователь admin может не иметь прав на удаление зданий с контроллерами
- Backend возвращает 403/401, но frontend не обрабатывает

- Предлагать удалить связанные данные или отменить операцию

### 3. Реализовать безопасное удаление
Варианты:
- **Мягкое удаление**: добавить поле `deleted_at` вместо физического удаления
- **Каскадное удаление**: удалять связанные контроллеры и метрики
- **Архивация**: перемещать данные в архивную таблицу

### 4. Добавить проверку перед удалением
```javascript
// Пример проверки
async function canDeleteBuilding(buildingId) {
  const controllers = await getControllersByBuilding(buildingId);
  const activeControllers = controllers.filter(c => c.status === 'online');
  
  if (activeControllers.length > 0) {
    return {
      canDelete: false,
      reason: `Building has ${activeControllers.length} active controller(s)`,
      suggestion: 'Deactivate or remove controllers first'
    };
  }
  
  return { canDelete: true };
}
```

## Обновление от 20 октября 2025

### Реализованные исправления

Проблема была успешно решена путём реализации следующих улучшений:

#### 1. Информативные диалоги ⚠️
Теперь при попытке удалить здание с контроллерами отображается детальный диалог:
```
⚠️ ВНИМАНИЕ: Невозможно удалить здание

Невозможно удалить здание с привязанными контроллерами

Привязанные контроллеры:
  • Контроллер #1 (CTRL-NAV-001) - online

Выберите действие:
• ОК - Удалить здание вместе с контроллерами и всеми метриками (НЕОБРАТИМО)
• Отмена - Отменить удаление
```

#### 2. Каскадное удаление 🔥
Реализована новая функция `deleteBuildingCascade()` которая:
- Последовательно удаляет все привязанные контроллеры
- Логирует прогресс и собирает ошибки
- Удаляет здание после удаления контроллеров
- Показывает детальный результат операции

#### 3. Улучшенная обработка ошибок
- Исправлена проверка поля `error` вместо `message`
- Добавлена специальная обработка ответа 400 с `errorData.controllers`
- Toast уведомления для всех сценариев (успех/предупреждение/ошибка)
