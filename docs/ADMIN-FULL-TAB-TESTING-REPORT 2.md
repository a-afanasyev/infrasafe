**Первоначальная ошибка:**
```
Ошибка загрузки данных: HTTP error! status: 500
```

**Диагностика:**
```bash
# Запрос к API возвращал:
{
  "error": {
    "message": "Failed to fetch water sources: column \"source_id\" does not exist",
    "status": 500
  }
}
```

**Причина:**
- В файле `src/utils/queryValidation.js` была ошибка в whitelist колонок
- Для `water_sources` и `heat_sources` использовалось `source_id`
- В реальной схеме БД используется просто `id`

**Исправление:**
```javascript
// БЫЛО (строка 62):
water_sources: [
    'source_id', 'name', 'type', ...
],

// СТАЛО:
water_sources: [
    'id', 'name', 'source_type', 'capacity_m3_per_hour', ...
],
```

**Результат после исправления:**
```json
{
  "data": [],
  "pagination": {
    "page": 1,
    "limit": 10,
    "total": 0,
    "totalPages": 0
  }
}
```

✅ Вкладка теперь загружается корректно, показывает "Нет данных" (таблица пустая)
