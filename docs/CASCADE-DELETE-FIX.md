# 🔧 ИСПРАВЛЕНИЕ: Каскадное удаление зданий с контроллерами

**Дата**: 20 октября 2025  
**Проблема**: Невозможно удалить здание с контроллерами даже принудительно  
**Решение**: Улучшена функция каскадного удаления с удалением метрик

---

## 🐛 ПРОБЛЕМА

### Описание
При попытке каскадного удаления здания с привязанными контроллерами операция завершалась ошибкой:

```
Ошибка: Невозможно удалить контроллер с привязанными метриками
```

### Причина
Система имеет защиту на **двух уровнях**:

1. **Здание → Контроллер**
   - Здание нельзя удалить, если к нему привязаны контроллеры
   - Ошибка: `BUILDING_HAS_CONTROLLERS`

2. **Контроллер → Метрики** ⚠️ (НЕ УЧИТЫВАЛОСЬ)
   - Контроллер нельзя удалить, если у него есть метрики
   - Ошибка: `CONTROLLER_HAS_METRICS`

### Иерархия зависимостей
```
Здание
  └─ Контроллер #1
      └─ Метрика #1
      └─ Метрика #2
      └─ Метрика #3
  └─ Контроллер #2
      └─ Метрика #4
      └─ Метрика #5
```

**Старая логика** (НЕ РАБОТАЛА):
```
1. Попытка удалить Контроллер #1
2. ❌ Ошибка: У контроллера есть метрики
3. ❌ Здание не удалено
```

---

## ✅ РЕШЕНИЕ

### Новая логика каскадного удаления

```
1. Для каждого контроллера:
   1.1. Получить все метрики контроллера
   1.2. Удалить все метрики
   1.3. Удалить контроллер (теперь без метрик)
2. Удалить здание (теперь без контроллеров)
```

### Код исправления

**Файл**: `public/admin.js`  
**Функция**: `deleteBuildingCascade()`  
**Изменения**: +60 строк

```javascript
async function deleteBuildingCascade(buildingId, controllers) {
    let deletedControllers = 0;
    let deletedMetrics = 0;  // ← НОВОЕ
    let errors = [];

    // Для каждого контроллера: удаляем метрики, затем контроллер
    for (const controller of controllers) {
        
        // ШАГ 1: Получаем метрики контроллера ← НОВОЕ
        const metricsResponse = await fetch(
            `/api/metrics?controller_id=${controller.controller_id}&limit=10000`,
            { headers: { 'Authorization': `Bearer ${token}` } }
        );
        
        if (metricsResponse.ok) {
            const metricsData = await metricsResponse.json();
            const metrics = metricsData.data || [];
            
            // ШАГ 2: Удаляем все метрики ← НОВОЕ
            for (const metric of metrics) {
                const deleteMetricResponse = await fetch(
                    `/api/metrics/${metric.metric_id}`,
                    { method: 'DELETE', headers: { 'Authorization': `Bearer ${token}` } }
                );
                
                if (deleteMetricResponse.ok) {
                    deletedMetrics++;
                }
            }
        }
        
        // ШАГ 3: Теперь удаляем контроллер (уже без метрик)
        const controllerResponse = await fetch(
            `/api/controllers/${controller.controller_id}`,
            { method: 'DELETE', headers: { 'Authorization': `Bearer ${token}` } }
        );
        
        if (controllerResponse.ok) {
            deletedControllers++;
        }
    }

    // ШАГ 4: Удаляем здание
    const buildingResponse = await fetch(
        `/api/buildings/${buildingId}`,
        { method: 'DELETE', headers: { 'Authorization': `Bearer ${token}` } }
    );
    
    // Успех!
    showToast(`✅ Здание, ${deletedControllers} контроллер(ов) и ${deletedMetrics} метрик успешно удалены`, 'success');
}
```
