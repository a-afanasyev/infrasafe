## 📦 Что нового

### 1. Сервис генератора тестовых метрик

**Зачем нужен**: Автоматическая генерация тестовых данных для зданий без ручного ввода

**Возможности**:
- 🎛️ Настройка диапазонов всех метрик через веб-интерфейс
- ⚡ Электричество: напряжение и сила тока по 3 фазам
- 💧 Вода: давление и температура (ХВС, ГВС)
- 🌡️ Окружение: температура воздуха, влажность
- 💦 Протечки: настраиваемая вероятность
- ⏰ Автоматическая генерация по расписанию (cron)
- 🐳 Docker контейнер для изоляции

**Как запустить**:
```bash
cd generator
docker-compose -f docker-compose.generator.yml up -d
```

**URL**: http://localhost:8081

**Документация**: `docs/GENERATOR.md`

---

### 2. Улучшенное удаление зданий в админке

**Что было**: При попытке удалить здание с контроллерами ничего не происходило

**Что стало**: 
- ⚠️ Информативный диалог с списком всех контроллеров
- 🔥 Возможность каскадного удаления (здание + контроллеры + метрики)
- 🛡️ Двойное подтверждение для безопасности
- 📊 Детальные уведомления о результате

**Пример**:
```
⚠️ ВНИМАНИЕ: Невозможно удалить здание

Привязанные контроллеры:
  • Контроллер #1 (CTRL-NAV-001) - online

Выберите действие:
• ОК - Удалить здание вместе с контроллерами и всеми метриками (НЕОБРАТИМО)
• Отмена - Отменить удаление
```

**Документация**: `docs/BUILDING-DELETION-IMPROVEMENTS.md`

## 🚀 Быстрый старт

### Генератор метрик

1. **Запустить контейнер**:
   ```bash
   cd generator
   docker-compose -f docker-compose.generator.yml up -d
   ```

2. **Открыть UI**: http://localhost:8081

3. **Настроить диапазоны**:
   - Ввести ID здания
   - Установить диапазоны для всех метрик
   - Нажать "Сохранить диапазоны"

4. **Запустить генерацию**:
   - Нажать "Выполнить генерацию один раз"
   - Или дождаться автоматической генерации по расписанию

### Удаление зданий

1. **Открыть админку**: http://localhost:8080/admin.html

2. **Войти**: admin / Admin123

3. **Выбрать раздел** "Здания"

4. **Удалить здание**:
   - Для зданий без контроллеров - удаление мгновенное
   - Для зданий с контроллерами - появится диалог с выбором действия

## ⚙️ Конфигурация

### Генератор

Файл: `generator/.env`

```env
PORT=8081
API_BASE_URL=http://host.docker.internal:3000/api
API_USERNAME=admin
API_PASSWORD=Admin123
GENERATOR_CRON=*/2 * * * *  # каждые 2 минуты
```

### Docker Compose

Файл: `docker-compose.generator.yml`

```yaml
services:
  generator:
    build: ./generator
    ports:
      - "8081:8081"
    volumes:
      - generator_data:/app  # персистентность данных
    extra_hosts:
      - "host.docker.internal:host-gateway"
```

## 🐛 Известные ограничения

### Генератор
- Нет UI для просмотра истории генерации
- Нет возможности изменить cron через UI
- Нет экспорта/импорта конфигурации

### Админка
- Каскадное удаление выполняется последовательно (может быть медленным при большом количестве контроллеров)
- Нет предпросмотра полного дерева зависимостей
- Нет возможности восстановить удалённые данные

## 💡 Советы

### Генератор
- Начните с небольших диапазонов для тестирования
- Используйте реалистичные значения для более качественных данных
- Проверяйте сгенерированные метрики через админку

### Удаление зданий
- Всегда читайте список контроллеров перед подтверждением
- Помните, что каскадное удаление необратимо
- Для production данных рекомендуется сначала отключить контроллеры вручную
